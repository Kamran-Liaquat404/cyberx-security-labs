Encrypt / Decrypt — Symmetric & Asymmetric

All crypto runs in your browser using the Web Crypto API. For real secure usage keep keys secret and back them up. This tool is educational; do not send sensitive secrets to untrusted services.

Symmetric (AES-GCM via password)
Uses PBKDF2 (SHA-256) with random salt and AES-GCM. Output is Base64 JSON package.
Asymmetric (RSA-OAEP)
RSA-OAEP (SHA-256). RSA can encrypt small plaintexts (size limited by key and padding). For large data use AES + RSA key wrap.
Utilities
Ready. All operations local to your browser.
Security note: AES uses random salt & IV. RSA private keys should remain secret — do not paste private PEMs into untrusted sites. This tool is for learning & safe demos.

Help & Key Management

Symmetric encrypt output format (Base64 JSON): contains salt, iv, alg and ciphertext. To decrypt supply the same passphrase.

Examples / Quicktips

  • To encrypt long text: use AES (Symmetric). Share passphrase securely.
  • To share securely with someone: generate RSA keypair, export their public key, use RSA to encrypt AES key (hybrid). This UI shows direct RSA for small text.
  • Always verify keys, backup private keys securely (offline).
